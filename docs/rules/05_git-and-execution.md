# 05. Git操作・実行計画

## 🌿 Git操作ルール

### ❌ 禁止
- **ブランチ作成は一切禁止**（Claude Codeがセッションを変えると読み取りにくいため）
  - 禁止コマンド: `git checkout -b` / `git switch -c` / `git branch <name>`
- ユーザーから明示的な指示がない限り、**現在のブランチを変更しない**

### ✅ 許可
- 現在のブランチ上での `git add`, `git commit`, `git push`
- `git status`, `git log`, `git diff` などの読み取り操作
- `git pull` による最新化

## 🚀 実行計画・タイムアウト対策
- タスクを細かいサブタスクに分割し、**各ステップで保存・コミット**しながら進める
- 長時間処理が見込まれる場合は、**チェックポイントを明示的に計画**してから実行
- タイムアウトが起こりうる原因を予測し、**対策を立てて実行**（対策の妥当性もチェック）
